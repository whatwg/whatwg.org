I&#39;m talking about large and complex intranet applications that provide a user interface for managing the actual product.<br>
<br>
The examples I gave are:<br>
HP performence center - it&#39;s an interface for performing load testing.<br>
Imperva SecureSphere - it&#39;s a security application. The interface is quite complex. You define your security policies , monitor your sites state and much more.<br>
<br>
I send those links because I think they demonstrate the typical level of complexity of such apps.<br>
The GUI is usualy quite complex but not as complex or speed dependant as lets say an IDE , photoshop or Microsoft Word.<br>
<br>
This type of apps share the same difficulty - they need to be webby (usually over WAN too) - but building web GUI is a nightmare - there is no good solution.<br>
<br>
Usually before creating such an application a vast research takes place as to the best GUI framework.<br>
The options considered are usualy HTML or Flex,javaFX or some other non HTML framework.<br>
<br>
All the applications I know chose HTML , with a server side GUI layer like JSF.<br>
The development was always hard and time consuming.<br>
<br>
