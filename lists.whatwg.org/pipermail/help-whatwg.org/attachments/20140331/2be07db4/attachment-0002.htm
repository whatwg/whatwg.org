<p class="MsoNormal">Hi,<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">Firstly, I realise HTML 5 has obsoleted the table axis attribute, and that this discussion relates to a superseded standard. – This discussion is academic.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">My question is simply around the purpose and utilisation of the table axis attribute when it was originally defined.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal"><b>The <a href="http://www.w3.org/TR/html4/struct/tables.html%23adef-axis">
table axis</a> attribute is defined as:<o:p></o:p></b></p>
<p class="MsoNormal"><b><span lang="EN">This attribute may be used to place a cell into conceptual categories that can be considered to form axes in an n-dimensional space. User agents may give users access to these categories (e.g., the user may query the
 user agent for all cells that belong to certain categories, the user agent may present a table in the form of a table of contents, etc.).<o:p></o:p></span></b></p>
<p class="MsoNormal"><span lang="EN"><o:p> </o:p></span></p>
<p class="MsoNormal"><span lang="EN">My interpretation of this is that the axis facilitates logical transforming of table data. Which would provide a foundation for filtering data, as well as dynamically redefining the table axis.<o:p></o:p></span></p>
<p class="MsoNormal"><span lang="EN"><o:p> </o:p></span></p>
<p class="MsoNormal"><b><span lang="EN">An example where I see this being useful, would be client side dynamic adaption of table content (e.g. filtering) and client side includes, such as using various table webpages (similar to a database schema and tables).<o:p></o:p></span></b></p>
<p class="MsoNormal"><b><span lang="EN">For example, an FAQ: “To administrate the Domain, log onto the domain controller: “, with the PDC domain controller’s IP address dynamically selected and inserted by the client from a separate page with a table.<o:p></o:p></span></b></p>
<p class="MsoNormal"><span lang="EN"><o:p> </o:p></span></p>
<p class="MsoNormal"><span lang="EN">I realise this is the design of a database (with the exception of client, rather than server, side inclusion); again, I am trying to understand the reason axis was originally standardised.<o:p></o:p></span></p>
<p class="MsoNormal"><span lang="EN"><o:p> </o:p></span></p>
<p class="MsoNormal"><b><span lang="EN">Have I understood the original ratification for the axis attribute?<o:p></o:p></span></b></p>
<p class="MsoNormal"><b><span lang="EN">If so, was a client-side inclusion technology, compatible with granular extraction of table data, ever created?<o:p></o:p></span></b></p>
<p class="MsoNormal"><span lang="EN"><o:p> </o:p></span></p>
<p class="MsoNormal"><span lang="EN">Again, I recognise this is not an aspect of the current standard. This is academic, to understand what technologies were conceived from a former standard.<o:p></o:p></span></p>
<p class="MsoNormal"><span lang="EN"><o:p> </o:p></span></p>
<p class="MsoNormal"><span lang="EN">Many thanks,<o:p></o:p></span></p>
