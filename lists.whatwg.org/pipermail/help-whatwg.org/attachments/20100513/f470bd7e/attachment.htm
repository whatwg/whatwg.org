<div><div><b>In short author says:</b><br><br>You are making a blog app and you have a form with some fields for creating a new post and a toolbar with &quot;actions&quot; like &quot;Save&quot;, &quot;Delete&quot;, &quot;Cancel&quot;.<br>
Our objective is to write the form in a way that doesn&#39;t require javascript, just plain old html form and submit buttons.<br>Since the action url is defined in the Form tag and not in each individual submit button, our only option is to post to a generic url and then start &quot;if...then...else&quot; to determine the name of the button that was submitted. Not very elegant, but our only choice, since we don&#39;t want to rely on javascript.<br>
<br>The only problem is that pressing &quot;Delete&quot;, will submit ALL the form fields on the server even though the only thing needed for this action is a Hidden input with the post-id. Not very big deal in this small example, but I have forms with hundreds (so to speak) of fields and tabs in my LOB applications that (because of requirements) have to submit everything in one-go and in any case this seems very inefficient and a waste. If form nesting was supported, I would at least be able to wrap the &quot;Delete&quot; submit button inside it&#39;s own form with only the post-id field.<br>
---</div><div><br></div><div><div>One answer was to re-design the app so users could delete items without opening them...</div><div>Another answer was to have the forms side by side on the page, but not nested. then use CSS to make all the buttons line up pretty. But... it is &quot;<span class="Apple-style-span" style="font-family: Arial, &#39;Liberation Sans&#39;, &#39;DejaVu Sans&#39;, sans-serif; font-size: 13px; border-collapse: collapse; color: rgb(68, 68, 68); line-height: 17px; ">too hackish. Plus in a very large page ... it&#39;s almost impossible to completely separate the elements&#39; screen position from their position in the document&quot;.</span></div>
