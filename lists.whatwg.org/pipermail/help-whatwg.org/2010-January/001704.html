> Hi,
> HTML 5 Specs for dt element says:
> 
> (1) - [quote]Contexts in which this element may be used: 
> In a figure element containing no other dt element children.[/quote]
> 
> So, only one dt is allowed as child of figure. Isn't it?
> 
> Now let's look at Specs for figure element:
> 
> (2) - [quote]The first dt element child of the element, if any, represents etc...[/quote]
> 
> Why *the first* in (2)?
> If it must be unique according (1) *the first* in (2) puzzled me. 

There are a number of examples of this kind of thing in the spec. They are 
cases where the spec is defining what should happen in browsers (or other 
user agents) when the author doesn't do what is required. Basically, 
they're error handling rules.
