Traditionally, recipes are sectioned ordered lists.  Your suggestion to 
add support for continued lists is a good one, considering that sections 
within lists can't be added in a compatible manner. There seem to be 
exactly three problems:
	1) it's a new feature (that needs to be evaluated based on how much its 
needed)
	2) it requires elements to be given identifiers (which is quite OK, 
even if usually used for sections and forms that are usefully linked to)
	3) The lists will still splintered into the sections (which is plain 
unavoidable)

Anyone see any serious problems with the proposal? Can we find a use 
case besides a menu?
