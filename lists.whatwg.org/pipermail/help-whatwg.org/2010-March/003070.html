As I began to respond, I was getting hung up on the verbiage "page" and thought "maybe this is really two questions":

1) Should there be a content-wide "wrapper" element?
2) Should it be called <PAGE>?

To answer #2, I don't think so. The purpose of HTML is define the structure of a page, including metadata, title and content. If a <page> element did exist, it would have to live at the highest level -- replacing or wrapping <html> -- to be semantically sound. Unless the spec is going to be changed to allow multiple pages served from one file, I don't think a <page> element makes sense.

For the first question, I appreciate your example, but don't think semantics plays into it. <body> already encompasses the page content and your example for a new tag seems to be purely for styling purposes. It doesn't enhance the meaning of the code, and that falls squarely into the realm of <div>.

Does that make sense?
