> On a browser that supports video but not Ogg

Then the browser does not support <video>. The spec mandates both <video>  
element support + ogg theora support. Else the implementation is  
incomplete.
It would be the same like have <img> without the UA not supporting any  
kind of image format.

But I agree though that we need an API to test UA capabilities, because  
the list of supported video and audio formats can grow, and have several  
different implementations in different states in different UAs, therefore  
hiding this information from the web page would result in more bloated  
javascript code doing more broken browser sniffing.

Just 2 cents.
