>> Ouch. Who is doing this and why?
> 
> Why ouch?

Because we’ve always operated under the assumption that devicePixelRatio is the mapping from CSS px to device pixels at 1:1 scale/zoom, and that it is constant for a particular display. The only change content would see is if the window is dragged to/from a high resolution screen.

There are other ways to query the page zoom.

I fear this will break existing content. I don’t think we’d ever want to change the behaviour.
