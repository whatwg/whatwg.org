Hi,

http://www.whatwg.org/specs/web-apps/current-work/#contenteditable  
needs some fixing. First of all, matching for the values "true" and  
"false" as case-insensitive. Furthermore, attribute values other than  
"true", "false" or the empty string "" cause the attribute to be  
ignored. Given the inheritence model of contenteditable="" this gives  
slightly different results.

   http://dump.testsuite.org/2006/contenteditable/

... has some tests to verify these things in Internet Explorer 7. Note  
also that this implicitly adds support for the "inherit" value we try  
to remove...
