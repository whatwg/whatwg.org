> HTML 5 should expose a native JSON parser for web content.

This seems self-contained enough that I really think it should get its own 
spec rather than being grafted onto HTML5. HTML5 is big enough as it is.

I recommend putting this forward as a proposal in the Web Apps group or 
the ES3.1/ES4 group. Presumably it would be relatively easy to define and 
so would not be a particularly difficult spec to edit should someone want 
to volunteer to do so.
