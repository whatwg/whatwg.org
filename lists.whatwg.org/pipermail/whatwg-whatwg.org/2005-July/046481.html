On Wed, 6 Jul 2005, fantasai wrote:
>
> The HTML spec currently does not define all common attributes on all 
> elements. For example, the i18n attributes are not defined to exist on 
> the <base> element (because it has no textual content). However, the Web 
> Apps draft currently applies all common attributes to all elements. Are 
> there any plans to tailor the list of valid attributes to specific 
> elements when not all common attributes make sense?

No. In practice it is much, _much_ easier to implement if all common 
attributes apply to all elements. (And indeed in practice that's exactly 
what many implementations do.)
