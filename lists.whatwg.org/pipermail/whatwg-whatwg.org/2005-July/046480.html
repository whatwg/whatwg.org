The HTML spec currently does not define all common attributes
on all elements. For example, the i18n attributes are not
defined to exist on the <base> element (because it has no
textual content). However, the Web Apps draft currently applies
all common attributes to all elements. Are there any plans to
tailor the list of valid attributes to specific elements when
not all common attributes make sense?
