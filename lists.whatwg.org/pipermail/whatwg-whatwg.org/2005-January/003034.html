On Sat, 29 Jan 2005, Matthew Raymond wrote:
> 
> For the three <select> control (3SC) problem

IMHO, there is no three <select> control "problem".

Using three <select> controls is an accessible solution, easy to 
understand, and unambiguous.

I do not think we need to try to convert people who are using three 
<select>s to using type="date" (or any other WF2 solution for dates). The 
benefits it would bring are minimal (basically, it would be prettier, in 
UAs that support a graphical calendar -- in UAs that don't the interface 
would probably be identical or merely localised).

"Don't fix things what ain't broke."
