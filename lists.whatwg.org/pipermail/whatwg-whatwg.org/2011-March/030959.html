Just to note, what the CSS WG most needs right now as far as I can see 
is not blue-sky ideas; there are plenty of those floating around. 
What's needed is firming up the basis for the existing specifications 
(e.g. doing something about the underdefined mess that is the CSS 2.1 
box model), the way HTML5 actually specified things that HTML4 
"specified".  Also writing tests.  Lots of tests.
