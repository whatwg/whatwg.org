I think it's definitely worth pursuing. But while I'm personally
not especially fond of accesskey, it's in the same class as some
other legacy markup constructs -- something we may all agree is
far less than ideal, but that is already widely used in existing
content. So we still need to have a more precise spec for it if we
want to have a hope of UAs handling it interoperably.
