Web Applications 1.0, 1.8. Miscellaneous[1]

# To ease migration from HTML to XHTML, UAs conforming to this
# specification must place elements in HTML in the
# http://www.w3.org/1999/xhtml namespace, at least for the purposes of
# the DOM and CSS.


Web Forms 2.0, 1.7. Terminology[2]

# Unless otherwise stated, XML elements defined in this specification
# are elements in the http://www.w3.org/1999/xhtml namespace, and
# attributes defined in this specification have no namespace. This does
# not apply to HTML as HTML does not support namespaces.


The possible contradiction would be of course that WA1 states that UAs 
should use namespaces for HTML and WF2 states that HTML doesn't have 
namespaces. It should probably be made more clear that the paragraph in 
WF2 applies to document authors and not to implementors (UAs).

[2]<http://whatwg.org/specs/web-forms/current-work/#terminology>
