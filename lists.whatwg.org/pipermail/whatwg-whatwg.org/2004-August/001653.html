Anne van Kesteren wrote:
> 2.2.6. The command element and commands
> 
> Why is the content model of the COMMAND element empty? It would be a lot 
> better imo if it was designed like BUTTON. That would also mean that the 
> LABEL attribute can be left out.

    The <command> button is intended to be used in situations where you 
don't want the menu to render at all on a legacy browser. Therefore, if 
it has text inside it, that text will be rendered when the user doesn't 
want it to be.
