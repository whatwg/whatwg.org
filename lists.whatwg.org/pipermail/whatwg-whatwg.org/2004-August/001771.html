On Wed, 18 Aug 2004 17:54:54 +0300, Mikko Rantalainen <mira at st.jyu.fi> wrote:

> I think that text input and "Browser..." control combination should
> be replaced with a single button. 

Don't you dare crippling my browser's UI!!! ;-)
Seriously, the lack of somewhere-to-type-a-path is the #1 reason why I
always find it so frustrating to work on a Mac. I paste and type into
file inputs every single day and one somewhat theoretical exploit
scenario isn't going to convince me it is bad UI. (I'm probably the
only person in the world to miss the old Opera implementation that
allowed the server to send a "value" attribute for file inputs and
simply asked the user to confirm the upload..)

Regarding the proof-of-concept in the Mozilla bug database
(http://bugzilla.mozilla.org/show_bug.cgi?id=57770) : it's hardly an
issue in Opera because Opera doesn't copy hidden characters. The
Moz-devs should focus on fixing that, then perhaps we should all
display selected entries in a "file-like" way with only path-less name
and icon when input focus is not in the field to make it clearer to
the user that this is about a *file* upload. That should do.
