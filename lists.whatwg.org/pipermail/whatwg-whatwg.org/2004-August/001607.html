Greetings,

DH> Could the use of the 'menu' element cause problems/confusion for
DH> browsers that support HTML 2.0 
DH> <http://www.rfc-editor.org/rfc/rfc1866.txt>?

More for the user than the html 2.0 browser. The browser is likely to
ingore what it does not understand, potentially rendering an unusable
menu.

The working group's main concern for graceful degradation is UAs that
support html4/xhtml1, in particular MSIE 6.

The 'menu' element exists in html 3.2, 4.01 trans and xhtml 1. trans
as well. It was depreciated in html4 (IMO) because UAs did not follow
the spec-recommended rendering in favor of an unordered-list-like
rendering.

In WAML, the 'menu' element is being pulled from the ashes, dusted off
and updated.
