Some additional comments and questions:


1.7. Miscellaneous

# To ease migration from HTML to XHTML, UAs conforming to this
# specification must place elements in HTML in the
# http://www.w3.org/1999/xhtml namespace.

How can HTML have a namespace? HTML doesn't have namespaces. Is this 
perhaps the way some UAs work? They don't really have knowledge of SGML, 
but create some kind of DOM structure of HTML documents and apply 
styling and scripting upon it? (If so, maybe that should be described a 
bit.)

Maybe other things should be added as well. Example: UAs should render 
|<br />| as |<br>| and not as |<br>\n>|.


2.2.6. The command element and commands

Why is the content model of the COMMAND element empty? It would be a lot 
better imo if it was designed like BUTTON. That would also mean that the 
LABEL attribute can be left out.
