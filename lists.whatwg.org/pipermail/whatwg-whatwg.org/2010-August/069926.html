Jeremy Keith:
> 
> The <hr> element is currently defined as "a paragraph-level thematic break." I think <br> could be defined as "a text-level thematic break."

That makes perfect sense. The only problem I see is existing content which relies on consecutive ‘br’s producing multiple line breaks, i.e. often a paragraph-like appearance. (Although ‘br’s were always intended to be collapsed, browsers never did this.)
