I checked and the values in CanvasDrawingStyles don't correspond with CSS
values so it should be OK to turn them into enums.
Only 'font' corresponds to an actual CSS value [1] and must not be changed.

The proposed 'textDecoration' property would follow CSS so it *could* be a
DOMString (even though I feel the IDL is cleaner with enums).

1:
http://www.whatwg.org/specs/web-apps/current-work/multipage/the-canvas-element.html#dom-context-2d-font
