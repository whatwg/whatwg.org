>   <h2> to <h6> are subsection headings when used in <body>, and
>   equivalent to <h1> when used in one of the section elements.

I have a partial implementation of the new heading model (or rather, a 
heading model which contains both <section> and <hn>) as an outliner. 
This is exactly the way my implementation works since it seems like the 
most sensible way to support HTML4 semantics and <section>. Multiple 
headings per section are considered to simply be multiple headings. None 
of the new stuff (<navigation>, <sidebar>, <header>, etc. is supported.)
