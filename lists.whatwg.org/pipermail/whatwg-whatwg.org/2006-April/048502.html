a successor for the RFC had already been approved. The successor  
draft is
http://www.ietf.org/internet-drafts/draft-ietf-ltru-registry-14.txt

Is it safe to expect Web Apps 1.0 to change its normative reference  
on language tags to the successor of RFC 3066?

(XML 1.0 has "or its successor" language regarding RFC 3066, so that  
won't be a problem.)
