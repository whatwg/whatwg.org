Blanchard, Todd wrote:
> What I want to know is:  if the "cloned" node has an id attribute, 
> and id is meant to be unique, then how do we resolve this conflict? 
> I mention this because bad html and duplicate id's in documents have 
> caused havoc when trying to build javascript applications.

The ID attributes need to be duplicated in such cases, that's what 
existing browsers do.  Although it may cause trouble with some scripts 
that use getElementById() and depend on there only being one instance, 
it is up to the markup author to fix that.
