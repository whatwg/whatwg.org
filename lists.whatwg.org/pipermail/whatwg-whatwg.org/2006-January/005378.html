Hi,

In the DOCTYPE name state[1], Anything else, the spec currently says:

>If the name of the DOCTYPE token is exactly the four letters "HTML", or 
>exactly the four letters "html", then mark the token as being correct.

I think that ["H"|"h","T"|"t","M"|"m","L"|"l"] should be correct. I browsers 
I'm aware of that do doctype switching will use their standards mode for all 
case combinations.

[1] http://www.whatwg.org/specs/web-apps/current-work/#doctype0
