>    Personally, I'm beginning to see the utility in using <link> to 
> assign shortcuts, as suggested by Derek Featherstone:

Well, there's nothing stopping UAs from assigning access keys based on 
rel="" values. I think it's a fine idea. Doesn't need to be 
user-configurable, either. Alt-Up, for instance, could always map to 
rel="up" (and when there is none, it could map to "take me up one 
directory", which is often the same thing).

None of this seems like something a spec need mention.
