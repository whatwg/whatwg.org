Frankly, the sheer number of different date and time input methods I came
across easily confirms the need for a standard widget date/time set.


> It's the same problem as with phone numbers, social security numbers
> and other formatted numerical date, and it has the same solution:
> post an example format. Example: MM/DD/YYYY. This is common
> throughout the web. I've even seen popup error messages showing
> format examples.

Most (all?) of the examples above are initially empty, so the trick
that someone (you?) mentioned at some point, of doing:

   <input type="date" value="DD/MM/YYYY" ...>

...would work fine. For more sophisticated apps, JavaScript can be used
to insert the example in legacy UAs. (Several of the forms above don't
even have an example at all, so it's not even critical that example text
be included.)
