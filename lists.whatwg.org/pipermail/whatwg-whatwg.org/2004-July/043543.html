On Tue, 13 Jul 2004, Matthew Raymond wrote:
>
> I wonder if it might not be possible to come up with a system for
> tracking suggestions for WHAT WG specs that are made by the public.

Several people have proposed this, so maybe we should indeed do this.

If someone wants to take responsability for setting up and maintaining a
Bugzilla, Wiki, FAQ, or other collaborative system for discussion,
information tracking, issue tracking, or whatever, let me know.

It should be noted, though, that I will be using the mailing list to go
through issues and respond to questions, so such a system would be in a
supportive role, not the primary issue tracking system. In particular, I
wouldn't be going through Bugzilla bugs marking them FIXED for each typo
that people report. I would, however, be happy to use a system that
summarised all the suggestions for a particular feature, when looking at
designing a feature. For example, when we added the combo box support, it
would have been nice to have one page listing all the use cases, all the
requirements, all the suggestions, and all the pros and cons of each
suggestion, as well as any supporting data people had reported on how UAs
handled various kinds of markup.

I would, however, encourage such a system to give links into the mailing
list archives for each piece of information, saying where it was from.


> It would also help people understand why something was rejected.

Yeah, that would be very useful.


> For instance, people could go to a web page that has a list of topics
> like "hiding legacy code", then click on the "<ignore>"  suggestion and
> see not only my idea for the <ignore> element, but Ian's reasoning for
> why it should not be included. People could then add their own thoughts
> on the matter.

I would rather their thoughts be primarily added to the mailing list, but
I agree that it would be great to also have an explanation on a site
somewhere (e.g. notes.whatwg.org) that explained all this.


So. If people want to do this, let me know, and I'll provide you with the
necessary basic technical support.

Cheers,
