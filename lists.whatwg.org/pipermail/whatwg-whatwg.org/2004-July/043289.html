On Tue, 6 Jul 2004, Hallvord Reiar Michaelsen Steen wrote:
>
> Perhaps add something like..
>
>     The "value" attribute may be used with date and time types to suggest
>     an initial date/time for any calendar widget the UA may show. If
>     omitted current date and time is used.

But do we really want the following:

   Birthday: <input type="date" name="bd">

...to default to today? Don't forget that if the user doesn't see the form
and just hits submit, there'll be no way to distinguish it from an
intentional pick of today.

What if "today" isn't in the allowed range?

Note how people wanted radio buttons to all be unchecked by default. This
is the same thing IMHO.
