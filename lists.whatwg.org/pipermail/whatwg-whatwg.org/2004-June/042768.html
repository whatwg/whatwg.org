> Ok now I'm really confused. It's like ".*" sometimes matches newlines

Nevermind, I had completely misread the spec.

If you have /m, ^ and $ match the edges of each _line_, not each string.
so /^.*$/m matches blank lines (such as "\n\n" in the example you gave,
Hallvord). So we don't want multiline at all. I've removed that from the
spec. Please pardon my momentary insanity. :-)
