The main problem would be IE's lack of styled XML support. (The only
XForms implementations I know of involve sending IE text/html content,
which according to the HTML working group must be parsed as HTML, not as
XML, and therefore can't contain namespaces anyway. So most of the XForms
content on the Web now is, ironically, invalid.)
