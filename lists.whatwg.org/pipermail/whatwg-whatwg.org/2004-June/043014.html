> Er, I don't see where it says that with equal q values the order of

The problem is that an en-gb resource matches the "en" that is listed
with the higher quality value in the Accept-Language header, so that
"en-gb" is listed with a lower priority doesn't matter.

The opposite isn't true, though: A "en" resource does not match "en-gb"
in the Accept-Language header.
