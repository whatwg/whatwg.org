>> Is "The user agent stops fetching the media data before it is completely
>> downloaded" really a good description for abort?  It can trigger during
>> the NETWORK_LOADED state, can't it?  Also, doesn't it only fire when the

"The user agent stops fetching the media data before it is completely
downloaded" implies that 1) it cannot occur after the media is
completely downloaded, and 2) it will fire if the user agent stops for
any reason, e.g., if the network connection fails or the server closes
the connection.  As far as I'm reading the normative portion of the
spec, neither of those points is true.  (Actually, this sentence also
seems to be normative?  At least, I don't see it set off as
informative by anything.)
