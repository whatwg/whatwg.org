> Are there any talks at the moment whether the type=date input will allow a user to enter a date manually or use the date picker or, should developers fall back to using a standard input field and add the date picker via JavaScript?

The purpose of <input type=date> is to allow the browser to supply the
user with a datepicker.  Opera, for example, already does this.

If you would like to show a datepicker to users of all browsers, use
feature detection (such as through the Modernizr script) to detect
whether or not the browser support date inputs, and if not, supply the
js version.
