First of all, an <output> element already exists :

http://www.whatwg.org/specs/web-apps/current-work/multipage/the-button-element.html#the-output-element
"The output element represents the result of a calculation."

What I have understood about it is that it is a sort of opposite of the 
the <input> element. In HTML 4, in order to display results of 
calculations, authors used to have an <input readonly id="output"> which 
was kind of absurd. The current <output> element seems to be here to 
fill this gap.

Is it enough for your needs ?
