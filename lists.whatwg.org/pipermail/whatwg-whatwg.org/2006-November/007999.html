It sees <noscript> only has conformance criteria. I would expect it to  
require some things from user agents as well. Such as not exposing the  
contents when scripting is enabled and vice versa. It should probably deal  
with disabling scripting after the contents of the <noscript> in question  
have been parsed as well. (Or perhaps some "initial scripting state" that  
you cache before you start parsing and keep all the way through.)
