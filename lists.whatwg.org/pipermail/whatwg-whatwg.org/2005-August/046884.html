Matthew Raymond wrote:
> The simple fact of the matter is that, with the version of HTML5 you
> currently propose, there is no way to submit HTML in the DOM without
> scripting.

Considering you need scripting for applications anyway, that is no 
problem. Note that the WHATWG is not about inventing new things, 
'contentEditable' and 'execCommand' have been here for years.

Desktop browsers are not likely to implement all kinds of fancy new 
stuff. They rather support things when it increases support for 
applications/sites out there.
