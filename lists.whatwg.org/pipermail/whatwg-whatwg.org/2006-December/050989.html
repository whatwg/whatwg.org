I think we should keep both systems. Things worth registering in the  
global namespace because they're common and not very likely to be  
misused (like class names currently listed in the spec: example,  
issue, note, warning, error, search, copyright), could be registered  
globally, things which are more specialized would be registered more  
restrictively under a profile.

What worries me most about the Wiki idea is that it'll make  
conformance to HTML5 a moving target that would need to be  
reevaluated each time a new class is added on the Wiki.
