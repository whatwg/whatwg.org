On Tue, 29 Mar 2005, Mikko Rantalainen wrote:
> 
> Banks that *require* that UA supports autocomplete don't really 
> understand the problem. (Or they understand the problem but don't want 
> to fix it, instead they simply try to hide the problem.)

Fair enough.


> WF2 shouldn't require UAs to support this feature. Just a note that some 
> institutions insanely want this feature is enough.

The spec has already been changed to not require support. It is currently 
a SHOULD, which means that while UAs should indeed support the feature (as 
they should with any feature, if they want to be considered fully 
compliant), there may be reasons (such as the user saying otherwise, e.g. 
through a pref or through market research) for not supporting it.
