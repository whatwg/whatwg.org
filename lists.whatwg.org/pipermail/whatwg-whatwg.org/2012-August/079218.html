Hi Michael,

in your case, the first control point is not the same as the last point of
the subpath.
The 'scale(2,1)' set up a different coordinate system. You can rewrite your
code from this:

ctx.lineTo(100, 100);
ctx.scale(2, 1);
ctx.arcTo(100, 100, 100, 200, 100);

to this:

ctx.scale(2, 1);

*ctx.lineTo(50, 100);*
ctx.arcTo(100, 100, 100, 200, 100);

Rik

PS
Doing the following doesn't' seem to work on chrome and ie9 because they
didn't implement the second radius parameter:

ctx.lineTo(100, 100);
ctx.arcTo(200, 100, 200, 200, 200, 100);
