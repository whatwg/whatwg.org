<p class=MsoNormal><font size=2 face=Arial><span lang=EN-US style='font-size:
10.0pt;font-family:Arial'>The MAP element should be transparent.  <o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span lang=EN-US style='font-size:
10.0pt;font-family:Arial'>If it contains only AREA elements, it is invisible
and it can belong to phrasing content; it is best placed alongside the IMG
element that uses it; even the <a
href="http://www.whatwg.org/specs/web-apps/current-work/multipage/the-canvas-element.html#the-map-element">example
provided</a> is <a
href="http://validator.nu/?doc=http%3A%2F%2Fwww.2a.pl%2F%7Ene01026%2Fimagemap.htm">invalid</a>;
it is an undocumented deviation from <a
href="http://www.w3.org/TR/html401/sgml/dtd.html#special">version 4</a>
(content model deviations are undocumented).  AREA elements should not
belong to any category; they can appear only as immediate children of the MAP
element (otherwise they are ignored per HTML4 so we can safely make them
invalid).<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span lang=EN-US style='font-size:
10.0pt;font-family:Arial'>If it contains anything else, it must contain flow
content for legacy reasons; therefore, it must belong to flow content.  <o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span lang=EN-US style='font-size:
10.0pt;font-family:Arial'>(I am aware of the fact that AREA elements are
slightly deprecated.)<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span lang=EN-US style='font-size:
10.0pt;font-family:Arial'>Editorial: The title of <a

The canvas element — HTML 5</a> should be changed to something more
appropriate — the page displays also subsection 4.8.12.<o:p></o:p></span></font></p>
