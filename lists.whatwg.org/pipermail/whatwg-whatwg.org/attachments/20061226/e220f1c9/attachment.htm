<P><FONT SIZE=2 FACE="Verdana">I'm wondering if you collectively would consider adding the following to the spec; a recommendation that clients offer two &quot;modes&quot;; one mode being for users where the spec works as currently envisioned. The second mode would be for web developers and would generate errors for invalid markup as opposed to generating no errors (Ian had said it was preferred to not generate errors for invalid markup to ensure users were willing to use browsers based on the newer spec; a dual mode would give the best of both worlds.)</FONT></P>

<P><FONT SIZE=2 FACE="Verdana">If all clients had such a dual mode, I think it would be much more likely that web developers would create valid markup.&nbsp; I just hope you guys can envision that being something mentioned and marked as &quot;SHOULD&quot; in the spec.</FONT></P>
