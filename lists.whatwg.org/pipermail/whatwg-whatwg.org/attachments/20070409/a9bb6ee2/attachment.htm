<div><br></div>
<div>Overall, the RFC was struggling with the issue that there is no
&#39;uniform&#39; naming of codecs;&nbsp; the namespace for codecs is
dependent on the container format, so products that do container
conversion have to have tables of code matches.&nbsp; ugh.&nbsp;
That&#39;s why the RFC is as it is.</div>
<div><br></div>
<div>The RFC suggests that updated information would be done with
RFCs, which is a little heavy.&nbsp; The RFC as written formally
applies to 3GPP files and 3GPP2 files, but the definitions are
applicable for all ISO-family files. </div>
<div><br></div>
<div>As you&#39;ll see below, 3GPP has also defined it for avc1 in
MP4-family containers, but no spec. or registration authority provides
a pointer.&nbsp; We might want to ask IANA whether we could add
something to the MIME registry.</div><span class="q">
