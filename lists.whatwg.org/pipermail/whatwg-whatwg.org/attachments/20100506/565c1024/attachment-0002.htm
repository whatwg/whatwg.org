<p class=MsoNormal>Is there any value in adding an "href" or
"uri" or similar attribute to the <cite> element to indicate a
location for a work (or information about the work) or, in the case of a URI,
an indicator that can be used as a reference programmatically?<o:p></o:p></p>

</div>

<div>

<p class=MsoNormal><o:p> </o:p></p>

</div>

<div>

<p class=MsoNormal><q> has a "cite" attribute, so it seems to
me that if we have a place to link to further information in <q> it makes
sense to do so in <cite>.  After all, whether an author quotes from
a reference (<q>) or merely discusses it without quoting (<cite>),
both of these would end up in a works cited in a traditional paper.
 Therefore, I think both should link (or refer) to somewhere.<o:p></o:p></p>

</div>
