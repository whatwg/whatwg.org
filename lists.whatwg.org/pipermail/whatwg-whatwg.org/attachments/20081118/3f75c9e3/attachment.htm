Pages communicate with their workers (dedicated) via <a href="http://www.whatwg.org/specs/web-workers/current-work/#the-queue">queue of events</a>&nbsp;. What happens if the queue gets more and more events queued (as a result of postMessage or timer callbacks) and the worker thread does not consume them fast enough?<div>
<br></div><div><ul><li>setInterval can skip posting a callback if the previously posted one was not yet consumed.&nbsp;<br></li><li>setTimeout is probably ok as it is but if the worker script adds them in a loop it can be a problem.<br>
</li><li>postMessage could somehow indicate a queue overflow and ignore the attempt to post a message if the queue length exceeds some specific threshold.<br></li></ul></div><div><br></div><div>Basically, the queue probably should have a limit on it and once the limit is reached, the queue-based operations should start to fail, optionally with some indication.</div>
