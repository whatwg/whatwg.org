<p class=MsoPlainText>Prior to r4177, the matching of tag names for exiting the
RCDATA/RAWTEXT states was done as follows:<o:p></o:p></p>

<p class=MsoPlainText><o:p> </o:p></p>

<p class=MsoPlainText>"...and the next few characters do no match the tag
name of the last start tag token emitted (compared in an ASCII case-insensitive
manner)"<o:p></o:p></p>

<p class=MsoPlainText><o:p> </o:p></p>

<p class=MsoPlainText>However, the current revision doesn't include any comment
on character casing in its discussion of "Appropriate End Tags." 
Similarly, certain tokenizer states require that you check the contents of the
"temporary buffer" against the string "script" but there is
no indication of whether or not to do this in a case-insensitive manner.<o:p></o:p></p>

<p class=MsoPlainText><o:p> </o:p></p>

<p class=MsoPlainText>In both cases, should this comparison be done in an ASCII
case-insensitive manner or not? It might be helpful to clarify the spec in both
places in either case.<o:p></o:p></p>
