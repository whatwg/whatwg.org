</font></blockquote><div> </div></div>There's actually a fairly major related problem here. We hide the fallback content by treating it as display:none. Currently Gecko has a huge bug where a display:none plugin does not load/run. This works out well for the video fallback case. If we fix that bug, then unless we do some special magic, plugin-based video fallback will run and play audio while the <video> element plays --- very bad.<br>
<br>People have already discovered that <video src="hellokitty.ogv"><object data="hellokitty.ogv"></object></video> plays the audio track twice in Firefox, and aren't happy about it, although it's what the spec seems to suggest. I'm not sure how to modify the spec in a sane way to fix this case, though.<br>
