Following on with the parallellism:<br>
<br>
style=&quot;...&quot;: inline styles, quite related to things like<br>
onclick=&quot;javascript:goBack();&quot; (inline script statements); this would<br>
be equivalent to the current property=&quot;&quot; and about=&quot;&quot;. There is no<br>
issue with them, just that I feel they are not enough. This solves<br>
some cases, but not all.<br>
<br>
class=&quot;...&quot; when used explicitly to tie with a CSS .class selector;<br>
relates to the usages of onclick=&quot;javascript:myFunction();&quot; (rather<br>
than doing all the work there, it hooks with a function defined<br>
somewhere else); and there is currently no equivalent for semantics.<br>
<br>
&lt;style&gt; and &lt;script&gt; are used to define document-wide styles and<br>
behaviors. Once again, we lack something to achieve this for<br>
semantics. Introducing a &lt;metadata&gt; element as suggested could be a<br>
solution (I&#39;d rather prefer &lt;semantics&gt;, but that&#39;s mostly a matter of<br>
taste), but if somebody has any better idea I&#39;d be glad to hear it.<br>
<br>
&lt;link rel=&quot;stylesheet&quot;&gt; and &lt;script src=&quot;...&quot;&gt; allow to import<br>
stylesheets and scripts that might be shared by several documents. I<br>
guess &lt;link&gt; could be used to import semantics as well.<br>
