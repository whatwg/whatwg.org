<div dir="ltr">Currently the spec doesn't say anything about the rendering of the <audio> element. Webkit makes <audio> without "controls" display:none by default, which seems reasonable, but it would be nice if spec recommended this behaviour. When "controls" is added, the element needs to be visible, so it will need an intrinsic size. Since the spec specifies the intrinsic size for <video>, shouldn't it do so for <audio> as well?<br>
