<p class=MsoNormal><font size=2 color=navy face=Arial><span lang=EN-US
style='font-size:10.0pt;font-family:Arial;color:navy'>There are multiple ways
to encapsulate heterogeneous data in documents: application resources (the &#8220;res&#8221;
URL scheme), file archives (jar, zip, shar, tar, whatever).&nbsp; Of those
possibilities, MIME is most Web-friendly IMHO, although the downside is that
the transmitted resource grows.&nbsp; This should not be a problem for small
resources and using it for big resources does not gain you anything, so you just
get it as a separate file.&nbsp; And, as has been previously discussed, &#8220;jar&#8221;
is not a protocol, it is a content type.<o:p></o:p></span></font></p>
