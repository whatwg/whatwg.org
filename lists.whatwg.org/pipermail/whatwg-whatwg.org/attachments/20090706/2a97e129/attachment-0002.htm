<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Allowing XML namespaces to be “inserted” into
HTML5 is a neat feature, but challenges its definition as a subset of XML. 
I don’t believe the XML 1.0 specification allows this.  The XHTML
specification would allow a module to be added on to it (in the form of a DTD
fragment), but in this case it would no longer be its own markup language, but
an add-on to HTML5.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p> </o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>Mixing markup languages also makes the DOCTYPE declaration pretty
much null and void.  What’s the point of a DTD if you’re going
to add in other DTDs?<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
