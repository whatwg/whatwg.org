I agree with Matthew. I think the mechanisms for accessibility need to 
be up to the individual platform, and there's no good generic way to 
specify this. On Mac OS X, the approach is to allow keyboard navigation 
of controls and links, and to use an external assistive program for 
more in-depth manipulation of the  UI without a mouse. Access keys are 
totally out of place in this design. It might be nice to have a 
clear-cut specification for their optional use, but I don't think it 
makes sense to make them mandatory.
