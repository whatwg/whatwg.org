>From HTML 4.01:

type = content-type [CI]
    This attribute specifies the content type for the data specified by data. This attribute is
optional but recommended when data is specified since it allows the user agent to avoid loading
information for unsupported content types. If the value of this attribute differs from the HTTP
Content-Type returned by the server when the object is retrieved, the HTTP Content-Type takes
precedence.

The type attribute is already optional and only indicative. It can claim that something is
different from what it actually is and just be ignored. To someone reading the markup, it's no
better than a comment.

A video element not only lets people reading the markup know that something should be a video, it
also should prevent anything else from being loaded. So if I embed a video from another site, and
they change it to a nasty applet because they weren't too happy about my hotlinking, the applet
wouldn't get loaded and executed.
