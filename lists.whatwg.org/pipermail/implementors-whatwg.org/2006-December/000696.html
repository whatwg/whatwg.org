For test cases whose expected DOM is XML-serializable, the equivalent  
XHTML5 document as Canonical XML is one possible format against which  
to compare.

For test cases that test if the parser catches errors, even having  
directories of cases where at least one error has to be found and  
directories where no errors may be found goes a long way, even if it  
isn't as proper as testing if the right error(s) were found.


Regardless of the format, I suggest licensing the test suite under  
the MIT license.
