<p class=MsoNormal>My personal implementation delivers identical results, and
my reading of the spec is that this is compliant output given the definition of
tree construction -- the second &lt;a&gt; tag gets foster parented under the
first &lt;a&gt; tag, because the first &lt;a&gt; tag is selected as the foster
parent element when the second &lt;a&gt; tag is encountered.<o:p></o:p></p>

<p class=MsoNormal><br>
Of course, it is not actually HTML-compliant output, because of nested
&lt;a&gt; tags, which are invalid.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>
