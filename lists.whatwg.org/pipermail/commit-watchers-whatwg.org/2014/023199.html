[c] (0) Don't worry, once a spec is in /TR/ it is completely STABLE. That's why the XML spec hasn't changed since it was first published, neither having new editions nor having new errata. However, this spec is not stable, so we'd better change in this arbitrary way to keep up to date with the unchanging XML spec that we matched before, and that we once again match now, but that did not change in between.
Fixing https://www.w3.org/Bugs/Public/show_bug.cgi?id=25461
